# Agent Hub Setup Guide

This document is for a new coding agent or future maintainer that needs to consume the central registry.

## Principle

Agent-specific files are adapters only. The source of truth is `.agent-hub`.

An agent should not define MCP credentials, service DB mappings, or duplicated long-form instructions in its own config. It should either:

1. Point MCP servers to `.agent-hub/bin/agent-mcp.js`, or
2. Use files generated by `.agent-hub/bin/agent-sync.js`.

## Initial Setup

From the workspace root:

```bash
cp .agent-hub/secrets/.env.example .agent-hub/secrets/.env
chmod 600 .agent-hub/secrets/.env
node .agent-hub/bin/agent-sync.js
node .agent-hub/bin/agent-validate.js
```

Fill `.agent-hub/secrets/.env` with local secrets. Never commit that file.

On this workspace, existing legacy configs can be imported without printing secret values:

```bash
node .agent-hub/bin/agent-migrate-secrets.js
```

Use `--force` only when intentionally overwriting `.agent-hub/secrets/.env`.

## Previewing Agent Installation

Before changing live agent config files, generate previews:

```bash
node .agent-hub/bin/agent-install.js --plan
node .agent-hub/bin/agent-install.js --dry-run
```

Preview files are written under:

```text
.agent-hub/generated/install-preview
```

Only after reviewing the preview should an agent be installed:

```bash
node .agent-hub/bin/agent-install.js --apply --agent codex
```

Apply one agent at a time.

## Adding an MCP Server

Edit `.agent-hub/registry/mcps.json`:

```json
{
  "servers": {
    "new-server": {
      "type": "stdio",
      "command": "${NODE20_BIN}/some-mcp-server",
      "args": ["--flag"],
      "env": {
        "API_TOKEN": "${NEW_SERVER_API_TOKEN}"
      }
    }
  }
}
```

Then add it to one or more profiles:

```json
{
  "profiles": {
    "work": {
      "servers": ["new-server"]
    }
  }
}
```

Add the secret name to `.agent-hub/secrets/.env.example` with an empty value, then put the real value in `.agent-hub/secrets/.env`.

Run:

```bash
node .agent-hub/bin/agent-sync.js
node .agent-hub/bin/agent-validate.js
```

Check a server resolves without launching it:

```bash
.agent-hub/bin/agent-mcp.js staging-mysql --check
```

Probe MCP protocol connectivity:

```bash
node .agent-hub/bin/agent-test-mcps.js
node .agent-hub/bin/agent-test-mcps.js staging-mysql gitlab
```

## Adding Instructions

Preferred path for new shared instructions:

```text
.agent-hub/instructions/<topic>.md
```

Then add it to `.agent-hub/registry/instructions.json`:

```json
{
  "path": ".agent-hub/instructions/<topic>.md",
  "required": true
}
```

Run sync and validation.

## Adding Skills

Preferred path:

```text
.agent-hub/skills/<skill-name>/SKILL.md
```

The skill should use the `SKILL.md` format:

```markdown
---
name: example-skill
description: Use when ...
---

Skill instructions...
```

Run sync and validation. The generated skill index appears at:

```text
.agent-hub/generated/skills/README.md
```

Agent-specific skill installation is adapter-specific. If the agent supports Codex-style skills, symlink or copy `.agent-hub/skills/<skill-name>` into that agent's skill directory.

## MCP Adapter Contract For New Agents

If the agent supports stdio MCP servers, configure each local MCP like this:

```json
{
  "command": "/Users/garvittyagi/Documents/dotpe_new/ai/.agent-hub/bin/agent-mcp.js",
  "args": ["server-name"]
}
```

For remote MCP servers, use the `url` from `.agent-hub/registry/mcps.json` directly if the agent supports remote MCP URLs.

If the agent needs one config file containing all MCPs, use the generated adapter file closest to its native format:

- Codex: `.agent-hub/generated/codex/config.mcp.toml`
- Claude: `.agent-hub/generated/claude/settings.mcp.json`
- OpenCode: `.agent-hub/generated/opencode/opencode.mcp.jsonc`
- Copilot: `.agent-hub/generated/copilot/mcp-config.json`

Known live config targets are listed in `.agent-hub/registry/agents.json`.

## Instruction Adapter Contract For New Agents

Use:

```text
.agent-hub/generated/instructions/AGENTS.md
```

or the generated agent-specific equivalent:

- Codex: `.agent-hub/generated/codex/AGENTS.md`
- Claude: `.agent-hub/generated/claude/CLAUDE.md`
- OpenCode: `.agent-hub/generated/opencode/AGENTS.md`
- Copilot: `.agent-hub/generated/copilot/copilot-instructions.md`

If the agent automatically discovers a root `AGENTS.md`, copy or symlink the generated file to that expected location after review.

## Safety Rules

- Do not commit `.agent-hub/secrets/.env`.
- Do not add raw tokens/passwords/API keys to registry files.
- Do not edit `.agent-hub/generated/*` manually.
- Keep prod MCPs read-only and approval-gated.
- Prefer `AGENT_HUB_PROFILE=safe-data` for routine code work.
